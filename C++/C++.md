2. register 变量

   ```c
   //C 语言中的register变量
   
   1. 使用局部变量时，把栈内存中的变量放到CPU 中去执行 ，然后进行读写操作。对于读写操作频率很高的变量，就需要把变量用 register 修饰，这样可以把变量请求放在寄存器中，加快读写效率，但是这是一个请求，不是强制性的；
     
     
    2. register 的使用限制🚫
     a. register 变量必须要被CPU 接受的类型，也就是变量必须是一个个的单值，并且长度小于等于int 的长度
     b. register修饰的变量放在 寄存器中，不存在 & 的操作
     c. 只有局部变量和形参才可以用 register 修饰
   
   //C++中的register 
     
     1. register 关键字只是为了兼容C语言，形同鸡肋，因为现代编译器完全可以对代码进行优化
     2. C++中可以对register 修饰的变量进行取地址操作，但是在取地址时，自动去掉 register 的修饰作用
     
     
   #include <stdio.h>
   
   int main(void)
   {
     register int a = 10 ;
     int * p = &a ;  //error: address of register variable requested
     
   
     return 0;
   }
     
   ```

   1. 重命名全局变量                     

      ```c
      //C 语言中可以对全局变量进行重复的定义，原因在于C编译器把多个同名的全局变量链接到全局数据区的同一个地址上去
      
      //C++中不允许定义多个同名的全局变量，二义性的存在不被接受
      
      #include <stdio.h>
      
      int main(void)
      {
        
      #if 1  //
        //C
        int g_num ;
        
        int g_num ;
        
      #endif

      }
      ```

### 5. 取结构体的地址，用指针获取结构体变量中的值

```c
#include <stdio.h>

typedef struct Person
{
    int a;
    char b;
    int  c;
    char  d;

}MyPerson;
int main()
{

    MyPerson p  = {'a',100,999,'b'};

    printf("sizeof(p) = %zd\n",sizeof (p));

    //得倒首地址

    MyPerson * pp = &p;
    printf("%p\n",(MyPerson*)&p);  //0x7ffee9f74aa8
    printf("%p\n",pp);

    //得倒结构体变量的d

    printf("struct-->d = %d\n",p.c) ;

    int get_d = *(int *)((char *)pp+8);
    /*
     *  *(int *)((char *)pp+8)
     * 
     * pp 取得结构体首地址
     * (char *)pp 这样就可以把步长定为一个字节
     * (char *)pp + 8 地址就偏移到
     * 
     * (int *)((char*)pp+8) 这样就可以决定取到多大的内存来使用
     * 
     * *(int*)((char*)pp+8) 最后的解引用  这样就可以得倒对应内存对应的值
     * 
     * 
     * 
     */
    printf("%d\n",get_d);



    printf("Hello World!\n");
    return 0;
}


```



### 6.动态库和静态库

```cpp
//库：一段编译好的二进制代码，加上头文件就可以提供使用
//使用时机：
	1.当需要用别人写好的代码，就需要把代码编译成库，加上头文件就可以使用了（这样不会知道源码）
  2.大型的文件，想减少编译的时间，就把成熟的代码部分编译成库，这样只需要在编译的时候LINK一下就好
    
//静态库：静态链接库（Linux .a ，windows .lib，Mac .a）
    静态库在直接编译的时候，直接拷贝一份到目标文件，这段代码在目标文件中也就不会改变了。
    优点：程序不需要再去依赖外部的库，可以直接运行；
    缺点：目标程序体积过大；若修改库文件就需要重新编译，不是很灵活
    
    生成静态库：
   	1.先生成对应的 .o 文件
    2.ar -rcs libxxx.o test1.o test2.o -static
    
    使用：
    gcc xxx.c -o app -I 头文件 -L 库的路径 -l库的名称 
    或者把 libxxx.a 库文件放到lib下面
    
    - static 让目标程序彻底独立起来
    用nm 查看库里面的内容
    
    
    
    
//动态库：动态链接库（Linux .so ,windows .dll，mac dylib)
    编译时不会被拷贝到目标程序，在程序运行时候会去链接对应的动态库文件
    优点：程序的体积小，而且同一份库文件可以被多个程序共用（共享库）
    			修改动态库文件也只需要重新编译动态库就好，这样不需要整个文件全部重新编译
    
    缺点：会过于依赖外部的库文件，导致程序运行的时候可能会发生找不到库的问题；库的版本问题也需要关注
    
    生成动态库
    1. 生成与位置无关的代码 gcc -fpic -c xxx.c -I 头文件
    2.gcc -shared -o libxxxx.so -I 头文件
    
```
